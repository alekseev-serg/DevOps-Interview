# DevOps-Interview Junior level

### Вопрос: Что такое DevOps  
**Ответ:** 
Это культура совместного разработки ПО, обеспечивающая сотрудничество между разработчиками и операционными специалистами для автоматизации процессов разработки, тестирования и развертывания программного обеспечения.   

---
### Вопрос: Вы набираете google.com в браузере. Расскажите как можно подробнее, что происходит в это время?  
**Ответ:**  
Прикладной уровень (Уровень 7):
Пользователь запускает браузер и вводит "google.com".
Браузер ищет соответствующий IP-адрес для домена "google.com" с помощью DNS.  
Представительный уровень (Уровень 6):
Браузер отправляет запрос на DNS-сервер для получения IP-адреса.  
Сеансовый уровень (Уровень 5):
Устанавливается сеанс с DNS-сервером для обмена данными.  
Транспортный уровень (Уровень 4):
Браузер использует протокол TCP для установки соединения с сервером Google.  
Сетевой уровень (Уровень 3):
IP-пакет с запросом отправляется к серверам Google через маршрутизаторы.  
Канальный уровень (Уровень 2):
Используется Ethernet для передачи данных через сетевые устройства.  
Физический уровень (Уровень 1):
Данные передаются по физической сети, например, через витую пару или беспроводное соединение.

### Вопрос: Как работает HTTPS?
**Ответ:**  
Шифрование:
Клиент и сервер согласовывают алгоритм шифрования и обмениваются публичными ключами.
Клиент шифрует данные с использованием публичного ключа сервера, и только сервер может расшифровать их с помощью своего приватного ключа.  
Идентификация:
Сертификат SSL/TLS удостоверяет подлинность сервера, проверяя его цифровую подпись.
Это позволяет клиенту убедиться, что он связывается с именно тем сервером, которого ожидает.  
Целостность данных:
HTTPS гарантирует, что данные не будут изменены или скомпрометированы во время передачи.

### Вопрос: Опишите общую архитектуру операционной системы.  
**Ответ:**  
Операционная система состоит из 4 компонентов:  
1) Ядро ОС. Управляет ресурсами компьютера. Планирует процессы.
2) Драйверы устройств. Позволяет ОС взаимодействовать с устройствами.
3) Пользовательский интерфейс. Обеспечивает взаимодействие пользователя с ОС.
4) Прикладные программы. Программы, которые пользователи запускают для выполнения различных задач

### Вопрос: В чем преимущества контейнеров?
**Ответ:**
Контейнеры используют общее ядро с ОС и не используют собственные уникальные ОС.
Контейнер гораздо меньшая логическая еденица чем ВМ и тратит гораздо меньше ресурсов.
При одновременном размещении, больше запустится контейнеров, чем ВМ.
Скорость использования. Создаются и уничтожаются значительно быстрее чем ВМ.

### Вопрос: Какова файловая структура в Linux (UNIX) системах, расположенных в /etc, /dev, /proc, /sys, /lib, /var
**Ответ:**  
/etc - содержит конфигурационные файлы системы  
/dev - содержит файлы устройств
/proc - виртуальная файловая система, представляет состояние ядра и запущенных процессов
/sys - виртуальная файловая система, представляет информацию о состоянии ядра и подключенных устройствах
/lib - библиотеки, необходимые для программ
/var - содержит логи и временные файлы.

### Вопрос: В чем разница между soft и hard symlink?
**Ответ:** 
Soft symlink (symbolic link) содержит путь к файлу, в то время как hard symlink содержит ссылку на inode файла. 
В случае удаления исходного файла, soft symlink не будет работать, в то время как 
hard symlink сохранит доступ к данным.

### Вопрос: Как работают file permissions, зачем директории права исполнения (+x)?
**Ответ:** 
Разрешения файлов позволяют определить, кто может читать, записывать и выполнять файл
Право на выполнение для директоии позволяет пользователю выполнять действия внутри директории, например переход в
поддиректории. Если этого права не будет, то это может предотвратить доступ к содержимому директории.

### Вопрос: Что такое zombie process?
**Ответ:**  
Zombie process - это процесс, завершивший выполнение, но еще не удаленный из таблицы процессов. 
Он остается в этом состоянии, чтобы родительский процесс мог получить информацию о завершении дочернего процесса.

### Вопрос: Что такое swappiness?
**Ответ:**  
Это свойство ядра Linux, которое определяет, как часто система будет использовать пространство подкачки. Подкачка может 
иметь значение от 0 до 100. Проверить текущее значение cat /proc/sys/vm/swappiness. Для настольного ПК норма 60, для 
сервера желательно свести к минимуму - 10 - sudo sysctl vm.swappiness=10

### Вопрос: Что такое inode?
**Ответ:**  
Это структура данных в системах файлов, которая содержит метаданные о файле или каталоге, такие как разрешения, владелец, 
тип файла, время доступа и модификации, размер файла и ссылки. Каждый файл на файловой системе имеет свой собственный
inode, который используется для управления файлами и каталогами.

### Вопрос: Расскажите поэтапно процесс загрузки Linux с момента включения питания компьютера.
**Ответ:**  
1) Считывание BIOS при включении, который ищет устройства загрузки
2) BIOS передает управление на загрузчик GRUB
3) Загрузчик позволяет выбрать какую ОС загружать
4) Выбранное ядро загружается в оперативную память. Ядро инициализирует аппаратное оборудование и запускает процессы
5) systemd запускает системные службы и процессы
6) Инициируется процесс для входа в систему пользователем
7) Пользователь входит в систему и начинает работу

### Вопрос: Как убить процесс? Какие есть типы сигналов?
**Ответ:**  
Чтобы убить процесс в Linux используется команда kill в сочетанием PID процесса.
1) SIGKILL - 9 - насильственное завершение процесса kill -9 PID
2) SIGTERM - 15 - по умолчанию для kill, сигнал завершения процесса

### Вопрос: Что такое модель OSI, TCP/IP?
**Ответ:**  
Модель OSI (Open Systems Interconnection) - это сетевая модель, 
которая описывает семь уровней для передачи данных от устройства к устройству в сети компьютеров.

TCP/IP (Transmission Control Protocol/Internet Protocol) - это семейство протоколов, которые используются для с
оединения устройств в интернете. Он состоит из четырех уровней - сетевого уровня, транспортного уровня, у
ровня приложений и уровня доступа к сети.

### Вопрос: Для чего нужны network masks?
**Ответ:**  
Network masks используются для определения, какая часть IP-адреса относится к сети, а какая к устройству в этой сети. 
Они позволяют определить, какие узлы находятся в одной локальной сети, а какие в другой, управлять 
маршрутизацией данных и обеспечивать безопасность сети.

### Вопрос: Что такое коллизия? Почему возникает?
**Ответ:**  
Коллизия - это ситуация, когда два или более объекта пытаются занять один и тот же ресурс или местоположение 
одновременно. Коллизии могут возникать из-за конкуренции за доступ к ресурсам, несогласованности в работе 
различных частей системы или неправильного использования ресурсов.
